Firmware-Code from Arduino IDE


      /*Turns ons 3 LEDS one after each other, each turning on for 1 second before turning off. 
        This is adapted from the example code in the public domain.
      */
      // the setup function runs once when you press reset or power the board
      void setup() {
        // initialize digital pin LED_BUILTIN as an output.
        pinMode(13, OUTPUT);
        //initialize the pin 12 as another output
        pinMode(12, OUTPUT);
        //initialize the pin 11 as the last output
        pinMode(11, OUTPUT);
      }

      // the loop function runs over and over again forever
      void loop() {
        digitalWrite(13, HIGH);  // turn the first LED on
        digitalWrite(12, LOW); // the second LED remains off
        digitalWrite(11, LOW); // the third LED turns off
        delay(1000);                      // wait for a second
        digitalWrite(13, LOW);   // turn the first LED off
        digitalWrite(12, HIGH); // the second LED turns on
        digitalWrite(11, LOW); // the third LED remains off
        delay(1000);                      // wait for a second
        digitalWrite(13, LOW);   // first LED remains off 
        digitalWrite(12, LOW);  // the second LED turns off
        digitalWrite(11, HIGH); // the third LED turns on
        delay(1000); //wait for a second
      }

Question 2

Reading the Arduino Uno Rev3 tech specs, I see that the current per I/O is a recommended 20mA. However, going into the microcontroller (ATmega328P) specs, I see that the chip wide limit is 200mA. Assuming that I am using pins 2-13 (as 0,1 are only for RX and TX), and 12 pins at 20mA, that makes the total current to be 240mA, this is way higher than the current limit and would be too high. If I want to stay at 200mA for the entire chip, then the max I could go is 10 pins at each 20mA. To be safe, I would probably do 9 pins to make sure I don’t go over the 200mA maximum. This would result in a 180mA current draw.
